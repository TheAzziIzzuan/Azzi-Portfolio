CyberStrike

Chrome Extension for Automated Web Vulnerability Scanning & Penetration Testing

Overview

A dual-component security project combining a Chrome browser extension for automated web vulnerability scanning with a deliberately vulnerable PHP/MySQL test website for hands-on security training. The extension provides 6 OWASP-aligned scanning modules covering the most common web application vulnerabilities, while the test website serves as a controlled environment to validate each scanner's detection capabilities.

Dual-Component Architecture

Chrome Extension (Manifest V3)

Tabbed popup UI with 6 scanning modules, a background service worker for real-time network monitoring, and Acorn.js for JavaScript static analysis.

  • popup.html — Main UI with 6 tabbed scanning modules
  • popup.js — Core scanning engine (~1,100+ lines)
  • background.js — Network interception & SSRF rules
  • common.txt — 4,700+ path wordlist for brute-forcing
  • acorn.js — JavaScript parser for code analysis

Vulnerable Test Website (PHP/MySQL)

Deliberately insecure web application designed to be detected by the extension's scanning modules. Hosted locally via Node.js/Express.

  • login.php — SQL Injection via GET parameters
  • signup.php — Credentials exposed in URL
  • profile.php — IDOR — unvalidated user ID in query
  • download.php — File download without sanitization
  • admin.php — Hardcoded credentials + SSRF
  • public_api.php — Exposed DB credentials via API

6 Scanning Modules

General Website Scan Full reconnaissance — IP geolocation, DNS records, WHOIS data, SSL certificate details, and HTTP header analysis in one click.
Broken Access Control Directory brute-forcing with 4,700+ path wordlist, numeric iteration scanning, and file extension probing (.txt, .html, .php).
IDOR Scanner Insecure Direct Object Reference testing — iterates through user IDs, file paths, and resource identifiers to detect authorization bypass.
Cryptographic Failures Detects hardcoded API keys (Google, Stripe, AWS), secret tokens in headers, and sensitive data transmitted over plaintext HTTP. CWE-319, CWE-321, CWE-523.
Injection Vulnerabilities Tests for 12+ injection types with automated attack payloads — SQL, XSS, command, LDAP, XML/XPath, CRLF injection. CWE-89, CWE-79, CWE-77/78, CWE-90, CWE-91, CWE-113.
SSRF Detection Server-Side Request Forgery scanning — probes cloud metadata endpoints (AWS, GCP, Oracle), internal services, Kubernetes, Docker, and database ports.

Embedded Vulnerabilities in Test Website

  • SQL Injection — No Prepared Statements (CRITICAL) User input directly concatenated into SQL queries in login.php and signup.php. Login uses GET method exposing credentials in URL. No input sanitization or parameterized queries.
  • Server-Side Request Forgery / SSRF (CRITICAL) admin.php accepts arbitrary URLs via GET parameter and fetches them with file_get_contents() — enables access to internal services and cloud metadata endpoints.
  • Insecure Direct Object Reference / IDOR (HIGH) profile.php passes user ID directly into SQL query without authorization checks. Any user can access any profile by changing the ?id= parameter.
  • Hardcoded Admin Credentials (HIGH) Admin username and password hardcoded in plain text in admin.php. The public_api.php endpoint exposes database host, user, password, and admin credentials via JSON response.
  • Sensitive Data in Cookies (HIGH) User role, ID, and a base64-encoded "secret" stored in cookies without Secure or HttpOnly flags in mainpage.php. Cookie-based authentication with no session management.
  • Credentials in URL / GET Method (MEDIUM) Both login and signup forms use GET method, exposing username and password in the URL, browser history, server logs, and referrer headers.

Real-Time Background Monitoring

The extension's background service worker continuously monitors network traffic using Chrome's webRequest API and declarativeNetRequest rules:

Cleartext Credential Detection Intercepts HTTP requests containing sensitive fields (username, password, token, apikey) in GET/POST data. Flags login vs. signup contexts. CWE-319 / CWE-523.
Hardcoded Secret Detection Inspects response headers (Set-Cookie, Authorization) for leaked secrets, API keys, and authentication tokens in real-time. CWE-321.
SSRF Protection Rules Declarative net request rules blocking requests to internal IPs (127.0.0.1, 192.168.x, 10.0.x, 169.254.x, ::1). Monitors sensitive path access patterns.

Automated Attack Payloads

After detecting injection vulnerabilities, the extension can auto-inject attack payloads into form fields and submit them:

  • SQL Injection (CWE-89) Payload: 1' OR '1'='1 — Classic authentication bypass injected into both username and password fields. Auto-submits the form after injection.
  • LDAP Injection (CWE-90) Payload: *admin)(|(uid=*)) — Exploits LDAP query structure to extract all user entries by manipulating the filter logic.
  • XML Injection (CWE-91) Payload: admin' or '1'='1 — Blind XPath injection targeting XML-based authentication to bypass login controls.
