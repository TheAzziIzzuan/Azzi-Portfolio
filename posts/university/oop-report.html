Project Type: 2D platformer game (team project)

Project Pitch

This is a team-built 2D platformer inspired by the pacing and feel of Mario and MapleStory. The goal is to create a clean, responsive side-scrolling experience with tight movement, meaningful item interactions, and enemies that keep the player engaged without overwhelming them. We are not just building a game; we are also building the engine layer that powers it, so the project shows both gameplay design and software architecture.

From a visitor perspective, this project demonstrates that I can design core gameplay loops, structure a game engine that supports reuse, and translate a report-heavy submission into a clear, playable concept. The presentation focuses on the experience first (movement, combat, items, progression), while still highlighting the engineering behind it.

Gameplay Focus

  • Side-scrolling platformer gameplay
  • Player movement, jumping, and collision handling
  • Items that affect player stats and score
  • AI enemies with simple movement behaviors
  • Multiple scenes (main menu, gameplay, pause, win/lose)

Gameplay is designed to be familiar but polished: fast movement, clear feedback when the player collects items or collides with hazards, and a level flow that encourages replay. The items and points system make each run feel different, while the menu and scene structure keep the experience organized and easy to navigate.

Engine Highlights

  • Layered architecture: game layer on top of a reusable engine layer
  • Factory-driven creation for entities, managers, and screens
  • JSON-based level configuration for faster iteration
  • Collision handling with specialized handlers
  • Centralized resource management (textures, assets)

The engine is structured so new levels, entities, and screens can be added without rewriting existing systems. JSON configuration makes level design quick to update, and the factory pattern keeps entity creation consistent. This means the game is easy to extend, which is important for future features like new enemies, power-ups, or UI screens.

My Role

I focused on the game flow and player experience. On the engine side, I implemented the Simulation LifeCycle Manager (to handle setup, resets, and state transitions) and the Sound Manager (to control music and effects). On the game layer, I built the core screens (Main, Pause, Play, Win/Lose), which define how the player enters, interacts, and exits gameplay. I also contributed to UI testing and tiled map design to ensure a consistent, readable game environment.
