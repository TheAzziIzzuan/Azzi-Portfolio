ICT1011P1F - Delivery Parcel Flip Detection System

Project Type: IoT Monitoring System

Project Overview

An intelligent IoT system designed to detect and monitor package handling during delivery. The system tracks acceleration and temperature in real-time to identify if parcels are flipped, dropped, or exposed to extreme temperatures, providing valuable data for improving logistics and package safety.

Key Features

  • Real-time acceleration monitoring using BMA250 sensor
  • Temperature tracking for temperature-sensitive packages
  • Dual-mode operation: Online (cloud-based) and Offline (local logging)
  • Customizable alert thresholds for shock and temperature
  • Live data visualization with charts and analytics
  • Data export functionality (CSV format)
  • Real-time notifications for anomalies

Hardware Components

  • TinyZero Board (Arduino-compatible)
  • BMA250 3-axis Accelerometer Sensor
  • TinyScreen Display with buttons
  • WiFi101 Shield for wireless connectivity
  • Real-Time Clock (RTC) for timestamp accuracy

System Architecture

  • Hardware Layer: TinyZero microcontroller reads sensor data and manages display/user interface
  • Communication Layer: WiFi connection to transmit data to cloud server
  • Backend Layer: Flask-based Python server to receive, process, and store sensor data
  • Analytics Layer: Web dashboard for real-time visualization and data analysis

Software Technologies

  • Arduino/C++ for IoT device firmware
  • Python Flask for backend server
  • HTML/CSS/JavaScript for frontend dashboard
  • Chart.js for data visualization
  • WiFi101 library for wireless communication
  • RTCZero library for real-time clock management

Monitoring Capabilities

  • Shock Detection: Detects sudden impacts and sudden movements (acceleration threshold alerts)
  • Temperature Monitoring: Tracks ambient temperature to identify temperature excursions
  • Historical Data: Stores timestamp, acceleration, and temperature readings
  • Alert System: Desktop notifications when thresholds are exceeded

Data Logging & Export

  • Data stored in CSV format with timestamp, acceleration, and temperature
  • Support for both cloud storage and local logging
  • Real-time data display on web dashboard
  • Export functionality for further analysis

User Interface

  • Embedded TinyScreen display on device for live status monitoring
  • Menu-driven interface for settings adjustment
  • Web dashboard for remote monitoring and analytics
  • Real-time graphical representation of sensor data

Potential Applications

  • Fragile goods delivery tracking
  • Temperature-sensitive package monitoring (pharmaceuticals, food)
  • Logistics quality assurance
  • Supply chain visibility and accountability
  • Insurance and damage claim verification

Technical Challenges & Solutions

  • Power Consumption: Optimized sensor reading intervals to maximize battery life
  • WiFi Connectivity: Fallback to offline mode for areas without network coverage
  • Data Accuracy: Calibrated sensor readings and noise filtering algorithms
  • Real-time Alerts: Implemented notification system for immediate issue detection
