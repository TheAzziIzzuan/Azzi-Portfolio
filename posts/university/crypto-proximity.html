Privacy-Preserving Proximity Detection

Secure Location Proximity Using Paillier Homomorphic Encryption — ICT2205/ICT2213 Applied Cryptography

Overview

A client-server application that allows users to check whether they are near a friend without either party revealing their exact location. The system uses Paillier homomorphic encryption to perform proximity computations entirely on encrypted data, while the server operates as a zero-knowledge message relay with no access to any location information.

Client-Server Architecture

Server — Zero-Knowledge Proxy

TCP socket server that handles authentication, friend management, and encrypted message forwarding. The server never sees, processes, or stores any location data.

  • server-code.py — Multi-threaded connection handler
  • User Authentication — SHA-256 hashed password storage
  • Public Key Registry — Distributes Paillier public keys
  • Message Forwarding — Routes encrypted protocol messages
  • Data Persistence — JSON-based user/friendship storage

Client — Cryptographic Engine

All cryptographic operations happen client-side. Generates Paillier keypairs, encrypts location data, and performs/interprets homomorphic proximity computations.

  • client-code.py — ~1,360 lines, full protocol implementation
  • menu.py — Text-based UI with proximity result display
  • 2048-bit Paillier Keys — Generated locally, never transmitted
  • Fernet Encryption — Password-derived key for private key storage
  • Threaded Listener — Async message handling during operations

Two Proximity Protocols

Grid-Based Proximity

Determines if two users are in the same grid cell (1000×1000 unit cells) using homomorphic subtraction and randomized linear combinations.

  1. User encrypts their grid cell coordinates with their own public key
  2. Encrypted coordinates are sent to the friend via the server
  3. Friend computes E(r1·(grid_x - my_x) + r2·(grid_y - my_y)) homomorphically
  4. Random blinding factor r3 added to prevent ciphertext pattern inference
  5. User decrypts: result = 0 means same cell, non-zero means different cell
E(a) + b = E(a + b)  —  Paillier additive property
c · E(a) = E(c · a)  —  Scalar multiplication
Result: E(r1·Δx + r2·Δy) + r3·E(0)

Euclidean Distance

Determines if two users are within a specified distance threshold using the squared Euclidean distance formula, computed entirely on encrypted values.

  1. User encrypts coordinates (x, y) and pre-computed norm ‖a‖²
  2. Friend computes ‖a-b‖² = ‖a‖² + ‖b‖² - 2(a·b) homomorphically
  3. Random bit s flips comparison direction; random factor r scales the result
  4. User decrypts and checks the sign to determine if within threshold
  5. Binary result only: within threshold or not (~3-5% approximation)
‖a-b‖² = ‖a‖² + ‖b‖² - 2(a·b)
s=0: r·(dist² - thresh²)  |  s=1: r·(thresh² - dist²)
Sign + s → nearby or not nearby

Security Properties

  • Zero Server Knowledge The server cannot determine user locations even with full database access. All proximity data is encrypted with user-owned public keys before transmission. The server only forwards opaque ciphertext without inspection.
  • Mutual Location Privacy Neither party learns the other's exact coordinates. The grid protocol reveals only same-cell or different-cell; the Euclidean protocol reveals only within-threshold or not. Random factors r1, r2, r3 (grid) and s, r (Euclidean) prevent any inference beyond the binary result.
  • HMAC Message Integrity Every request includes an HMAC-SHA256 tag computed with a PBKDF2-derived key (100,000 iterations, username as salt). The server and client verify integrity to detect tampering.
  • Replay Attack Prevention All messages carry timestamps. Recipients check timestamp freshness to reject replayed messages, preventing adversaries from re-sending captured protocol exchanges.
  • Private Key Protection Private keys are generated locally and never transmitted. They are stored encrypted with Fernet using a password-derived key (PBKDF2, 100K iterations), ensuring keys at rest are protected even if the filesystem is compromised.

Application Features

User Registration Generates 2048-bit Paillier keypair, stores encrypted private key locally, registers public key with server
Secure Login SHA-256 password hashing, PBKDF2-derived HMAC key, automatic key loading and decryption
Friend Management Add friends, view online/offline status, select friends for proximity checks
Location Updates Set coordinates (0-99999), stored locally only, never sent to server in plaintext
Grid Proximity Check Same-cell detection via homomorphic subtraction with randomized masking and blinding
Euclidean Distance Check Threshold-based distance comparison via squared distance formula with random sign flip
