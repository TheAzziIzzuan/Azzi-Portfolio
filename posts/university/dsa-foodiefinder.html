Data Structures & Algorithms - FoodieFinder

Last Updated: 11/02/2026

Project Type: Full-stack web application (team project)

Project Overview

FoodieFinder is a web application that helps users find halal and vegan food options across Singapore. It combines an interactive map, real-time search, restaurant reviews, rating filters, and walking directions — all powered by data structures and algorithms under the hood.

The project goes beyond a simple restaurant listing. It uses Dijkstra's algorithm to calculate nearest restaurants and walking routes, a Trie data structure for fast autocomplete search, QuickSort for filtering by ratings, and sequential search for review lookups. The data itself was collected by building custom web scrapers.

What It Does

  • Interactive Map: Displays halal restaurants, vegan restaurants, and MRT stations across Singapore using Leaflet.js with color-coded markers
  • Find Nearest Food: Uses your current location to find the 7 nearest halal and vegan restaurants plus the closest MRT station
  • Walking Directions: Calculates and draws the actual walking route from your location to any restaurant or MRT station
  • Search: Type a restaurant or MRT station name and get instant autocomplete suggestions
  • Filter by Rating: Filter halal or vegan restaurants by minimum rating
  • Reviews: Browse aggregated reviews with ratings, search through them, and add your own

Data Structures & Algorithms Used

Dijkstra's Algorithm: Used to calculate shortest distances from the user's location to all nearby restaurants and MRT stations. The algorithm builds a graph of nodes (user location + all markers) and edges (distances between them), then finds the optimal nearest locations. It's also used for walking route calculation through OpenStreetMap road networks.

Trie (Prefix Tree): Powers the search autocomplete feature. All restaurant names and MRT station names are inserted into the Trie at startup. When the user types, the Trie efficiently returns all matching entries by prefix, providing instant suggestions without scanning the entire dataset.

QuickSort: Applied to filter and sort restaurant results by rating. When users filter by a minimum rating, QuickSort efficiently orders the results for display.

Sequential Search: Used for searching through reviews and matching user queries against restaurant titles in the dataset.

Web Scraping

The restaurant data was collected by building custom web scrapers using Selenium and BeautifulSoup. The scrapers targeted WhereHalal.com to extract halal restaurant listings including name, address, food type, certification status, and ratings. The data was cleaned and validated using Python scripts that checked for missing values and invalid characters.

Tech Stack

  • Backend: Python Flask with SQLAlchemy for user reviews database
  • Frontend: HTML/CSS/JavaScript with Bootstrap and Leaflet.js for maps
  • Data: CSV datasets for halal restaurants, vegan restaurants, and MRT stations with geocoordinates
  • Scraping: Selenium WebDriver + BeautifulSoup for data collection
  • Pathfinding: OSMnx + NetworkX for real walking route calculation on OpenStreetMap
  • Database: SQLite for storing user-submitted reviews

Why This Matters

This project demonstrates how data structures and algorithms aren't just theory — they solve real problems. Dijkstra's algorithm makes location-based search fast and accurate. The Trie makes search feel instant. QuickSort handles filtering efficiently even with large datasets. And the web scraping pipeline shows how to collect, clean, and serve real-world data.

From a full-stack perspective, the project covers backend API design, database management, frontend interactivity, map integration, and data engineering — all in one cohesive application.
