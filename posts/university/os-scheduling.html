Operating System Project - CPU Scheduling Algorithms

Last Updated: 11/02/2026

Focus: Comparing different approaches to CPU process scheduling

Project Overview

This project explores multiple implementations of CPU scheduling algorithms, with an emphasis on optimizing key metrics like waiting time, turnaround time, and context switches. I implemented and compared five different scheduling approaches, each using different strategies to improve upon the basic Round Robin algorithm.

The core challenge in CPU scheduling is balancing fairness (all processes get CPU time) with efficiency (minimizing idle time and context switches). This project shows multiple solutions to that problem.

Five Scheduling Approaches

1. Priority-Based Round Robin with VLBT Detection (paper1.c) — This implementation adds priority levels to standard Round Robin. High-priority processes with very light burst times (VLBT) are executed immediately, reducing their wait time. Other processes follow a priority-adjusted time quantum system. The key innovation is detecting which processes are almost done and letting them finish first, reducing overall waiting time.

2. Dynamic Quantum Based on Max Burst Time (paper2.c) — Instead of a fixed time quantum, this algorithm calculates the quantum dynamically as 80% of the maximum burst time in the system. When all processes exceed the quantum, the quantum is increased to the maximum burst time. This adapts the scheduling to the actual workload characteristics.

3. Multi-Queue Light/Heavy Scheduling (paper3.c) — Processes are split into two queues based on whether their burst time is above or below the median. Light processes get a smaller time quantum (median), while heavy processes get a larger one (median of heavy tasks). This segregation allows each group to be optimized independently.

4. Improved Round Robin with Dynamic Mean-Based Quantum (paper4.c) — The time quantum for each round is calculated dynamically based on the mean burst time of processes currently in the ready queue. The formula is: TQ = (mean/2) + (mean/2)/burst_time. This provides finer-grained control and adapts as the queue changes.

5. Median-Based Time Slice Calculation (paper5.c) — This algorithm uses the median of remaining burst times in the ready queue to calculate the time slice. For initial scheduling, the median of processes arriving at the earliest time is used. This approach balances between light and heavy processes naturally through statistical measures.

Key Performance Metrics Measured

  • Waiting Time: How long a process waits before execution starts. Lower is better for responsiveness.
  • Turnaround Time: Total time from arrival to completion. Indicates overall efficiency.
  • Response Time: Delay until first execution. Critical for interactive systems.
  • Context Switches: Number of times the CPU switches between processes. Fewer is better to reduce overhead.

Why This Matters

Operating systems handle thousands of processes concurrently. A poorly designed scheduler can cause performance degradation, long wait times, and high CPU overhead from context switching. By comparing these five approaches, I explored how different heuristics (priority, burst time analysis, queue segregation, dynamic adjustment) affect real-world scheduling performance.

The implementations show that there's no one-size-fits-all solution. Different workloads benefit from different strategies. A workload with many short bursts benefits from VLBT detection, while a mixed workload might benefit from multi-queue segregation.

Technical Implementation

Each implementation is a complete C program that accepts user input for process arrival and burst times, simulates the scheduling algorithm, and outputs key metrics. The programs use dynamic time calculation, queue management, process state tracking, and statistical analysis to implement sophisticated scheduling logic.

The projects demonstrate proficiency in C programming, algorithm design, performance analysis, and system-level thinking about how operating systems manage computational resources.

Deliverables

Five complete C implementations with detailed comments, each handling real-world scheduling scenarios, calculating performance metrics, and providing insights into CPU utilization and process management efficiency.
