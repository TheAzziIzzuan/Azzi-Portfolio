Networking Project - Designing and Implementing a Scalable Enterprise Network

Last Updated: 11/02/2026

Course: INF1013 Computer Networks (AY2023/24 Trimester 2)

Project Overview

This project demonstrates the design and implementation of a fault-tolerant, scalable enterprise network using Cisco switches and routers. The network supports multiple departments with unique requirements, segregates traffic using VLANs, ensures high availability through redundancy protocols, and provides secure internet access.

The key challenge was building a network that balances security, performance, and cost-efficiency while allowing seamless failover and inter-VLAN communication. I collaborated with my team to design the topology, configure routing and switching protocols, and document the infrastructure.

Network Architecture

  • Core Switches: Two distribution switches (Layer 3) act as the network backbone with redundancy
  • Access Switches: Three Level access switches connect end-user devices to the core
  • Routers: Two routers provide internet connectivity and serve as ISP gateways
  • Departments: 6 VLANs for HR, Sales, Meeting Room, Technical, Finance, and Country Director Office

Key Design Principles Implemented

VLAN Segmentation: Each department is isolated in its own VLAN to enforce security policies, control access, and simplify network management. This allows the IT team to apply different rules and monitor each department independently.

EtherChannel (Link Aggregation): Two physical links between distribution switches are combined into a single logical channel, doubling bandwidth and providing redundancy. If one link fails, traffic automatically flows through the other without interruption.

Spanning Tree Protocol (STP): Prevents network loops by ensuring only one active path exists between devices. If the primary path fails, STP automatically activates a backup path, maintaining connectivity.

Hot Standby Router Protocol (HSRP): Provides transparent failover between routers. If the primary router goes down, the standby router takes over automatically without users noticing outages. Each VLAN can have different active/standby routers for load balancing.

OSPF Routing: Dynamically calculates the shortest paths between routers and subnets. If a link fails, OSPF recalculates routes in seconds, ensuring traffic continues to flow through available paths.

Port Address Translation (PAT): Allows all internal devices to share a small pool of public IP addresses. This saves costs, conserves addresses, and hides internal IPs from the internet, improving security against attacks like DDoS.

Technical Implementation

The project included configuring and documenting all device configs:

  • 2 Distribution Switches (Cisco Catalyst 3650) running VLANs, HSRP, and STP
  • 3 Access Switches (Layer 1, 2, 3) connected to end devices
  • 2 Routers (Cisco 2911) running OSPF, NAT, and PAT for internet gateway
  • IP addressing scheme across all subnets with proper documentation
  • Redundancy at every level (switches, links, routers)

Why This Matters

Enterprise networks are critical to business operations. Downtime costs money. This project shows that I can design networks that stay up even when components fail. By segmenting departments into VLANs, we improve security and performance. By adding EtherChannel, STP, and HSRP, we build resilience. By using OSPF, we make routing automatic and flexible. And by implementing NAT and PAT, we manage IP addresses efficiently and securely.

The documentation and configuration files in the project show that I understand not just the theory, but how to actually configure and troubleshoot real Cisco equipment.
