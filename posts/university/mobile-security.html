Mobile Security

Building a Trojanized Android App to Demonstrate Mobile Threats

Overview

A group project where we built a fully functional Android social media application that secretly contains multiple malicious payloads — hidden behind a legitimate-looking interface. The goal was to demonstrate how a trojanized app can abuse Android permissions and APIs to steal sensitive data without the user realising it.

On the surface, the app is an Instagram-style platform: users sign up, post photos and videos, browse an activity feed, request events, leave reviews, follow other users, and manage their profile. Underneath, the app silently extracts contacts, tracks GPS location, logs device sensor data, and can send SMS messages — all while appearing completely normal to the end user.

Two Faces of the App

What the User Sees

A polished social media app with Firebase authentication, a scrollable activity feed with posts and likes, a camera for capturing photos and recording videos, gallery-based media upload, an events system with admin approval workflow, user profiles with image/video grids, a follower system, push notifications, and a review/rating feature for events.

What Actually Happens

Behind the scenes, the app harvests the user's entire contact list and uploads it to Firestore, tracks their GPS coordinates on a timer and exfiltrates them to a remote database, runs a foreground service that logs accelerometer and gyroscope data to an external HTTP endpoint, and has the ability to send SMS messages programmatically — all triggered by normal-looking app interactions.

Hidden Malicious Features

  • Contact Exfiltration When the user grants the READ_CONTACTS permission (prompted with a social-engineering excuse like "find friends"), the extractContacts() method in ActivityFeedActivity silently reads every contact name and phone number via ContactsContract and bulk-uploads them to a Firestore collection.
  • GPS Location Tracking AdminEventReviewActivity requests ACCESS_FINE_LOCATION with a misleading dialog: "Enable location to see local events and bookings near you faster." Once granted, a Timer calls startStealingLocation() on a loop, feeding the user's latitude, longitude, and email into Firestore every few seconds.
  • Sensor Data Logging SensorLoggingService runs as a persistent foreground service, registering listeners on the accelerometer and gyroscope. Sensor readings are batched into JSON arrays and POSTed to an external HTTP endpoint — enabling motion profiling and activity inference on the attacker's server.
  • Silent SMS Sending SmsHelper uses Android's SmsManager to send text messages to arbitrary numbers programmatically. This could be weaponised for premium-rate SMS fraud, spam distribution, or further social engineering attacks against the victim's contacts.

Social Engineering Techniques

The malicious features are carefully disguised behind legitimate-sounding permission requests and normal UI flows:

  • Permission Justification Contact access is framed as "find friends." Location access claims to show "local events near you." The user sees a reasonable explanation and taps "Allow" without suspecting data theft.
  • Trigger Points Malicious code is triggered by normal user actions — opening the activity feed starts contact extraction, navigating to the event admin page starts location tracking. There's no visible indicator that anything abnormal is happening.
  • Foreground Service Camouflage The sensor logging service runs as a foreground service with a persistent notification — which on Android looks like a normal background feature of the app, not something suspicious.

Tech Stack

Android (Java) Native Android app using CameraX, ExoPlayer, RecyclerView, Glide, GestureDetector
Firebase Suite Auth (email/password), Firestore (data + exfiltrated data), Storage (media uploads)
Android APIs Abused ContactsContract, FusedLocationProviderClient, SensorManager, SmsManager
Admin Features User ban/unban system, event approval workflow, review moderation panel

Legitimate App Features

Reels & Camera CameraX integration for photo capture and video recording, gallery browser, ExoPlayer for video preview
Activity Feed Scrollable post feed with likes, comments, and swipe-to-refresh — modelled after Instagram
Events System Users request events (name, date, time, description), admins approve/reject/complete, users leave reviews with star ratings
Profiles & Social User profiles with image/video grid tabs, follower counts, post counts, logout, and user search

My Role

I worked on integrating the malicious payloads into the app's normal user flows — implementing the contact exfiltration logic that triggers on the activity feed, the GPS tracking timer embedded in the event review page, and the sensor logging foreground service. I also contributed to the legitimate front-end features to make the app convincing enough that a real user wouldn't suspect anything.
